# Контекстно-зависимые условия

Реализована проверка правила 1 (уникальность идентификаторов). Программа контролирует уникальность имен на двух уровнях:
1. Уникальность имени перечисления
2. Уникальность элементов внутри одного перечисления
$Примечание: при обнаружении дубликата, узел исключается из итогового AST$
# Структура AST

#### a. Описание типов узлов

В основе дерева лежит абстрактный класс AstNode, от которого наследуются конкретные узлы конструкции:
1. EnumDeclarationNode — корневой узел объявления перечисления.
		- Name (string): Имя перечисления.
		- Modifiers (List String): Список модификаторов (например, enum, class).
		- Elements (List EnumElementNode): Дочерние узлы — элементы перечисления.
2. EnumElementNode — узел конкретного элемента (константы) внутри перечисления.
		- Name (string): Имя элемента.

#### c. Формат вывода AST в программе

```
└── EnumDeclarationNode
    ├── name: "Day"
    ├── modifiers: ["enum", "class"]
    └── elements:
        ├── EnumElementNode (name: "Monday")
        └── EnumElementNode (name: "Tuesday")
```
